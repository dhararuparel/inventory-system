Database Constraints & Security Notes
  • Audit Logging: Every action processed inside the main app dashboard generates permanent record rows. Editing the audit log tables directly from this admin panel is restricted to safeguard data validation compliance.
  • Negative stock prevention: The database enforces an integrity check constraint: `CK_INVENTORY_QUANTITY_NONNEGATIVE`. Entering a negative count for a location's inventory level will trigger database-level errors and rollback the transaction.
  • Adding locations: You can add locations (e.g. Godown #3, Godown #2) in the "Locations" table. Once added, stock operations (In, Out) can immediately leverage them.
